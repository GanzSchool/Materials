A **literálok** a programozási nyelvekben olyan fix, közvetlenül a kódban megadott értékek, amelyeket változókhoz rendelünk vagy közvetlenül használunk műveletekben. A JavaScript-ben a literálok különböző típusú adatokat képviselnek, mint például számok, szövegek, logikai értékek, objektumok és tömbök. Ezek az értékek nem változnak a futás során, hacsak explicit módon nem módosítjuk őket (például objektumok és tömbök esetében).

### Literálok Típusai a JavaScript-ben

#### 1. Szám Literálok (Numeric Literals)

Számok közvetlenül a kódban megadhatók, legyenek azok egész számok vagy valós számok.

```javascript
let egészSzám = 42;        // Egész szám literál
let valósSzám = 3.14;     // Valós szám literál
let negatívSzám = -7;      // Negatív szám literál
let exponenciálisSzám = 1.2e3; // Exponenciális szám literál (1.2 × 10³ = 1200)
```

#### 2. Szöveg Literálok (String Literals)

Szövegek idézőjelek között vannak megadva. JavaScript támogatja a dupla (`" "`), egyszeres (`' '`) és template literálokat (backtick-ekkel: `` ` ` ``).

```javascript
let duplaIdézőjel = "Hello, World!"; // Dupla idézőjel literál
let egyszeresIdézőjel = 'Hello, World!'; // Egyszeres idézőjel literál
let templateLiteral = `Hello, ${name}!`; // Template literál, változó interpolációval
```

#### 3. Logikai Literálok (Boolean Literals)

Két lehetséges értékkel rendelkeznek: `true` (igaz) és `false` (hamis).

```javascript
let igaz = true;    // Boolean literál
let hamis = false;  // Boolean literál
```

#### 4. Specialitás Literálok

##### a. `null`

A `null` egy speciális érték, amely azt jelenti, hogy nincs érték. Gyakran használják arra, hogy szándékosan üres értéket jelöljenek.

```javascript
let nincsErtek = null; // Null literál
```

##### b. `undefined`

Az `undefined` azt jelenti, hogy egy változó deklarálva lett, de még nem kapott értéket. Automatikusan kerül hozzárendelésre bizonyos esetekben.

```javascript
let nincsHozzarendelve;
console.log(nincsHozzarendelve); // undefined
```

#### 5. Objektum Literálok (Object Literals)

Objektumokat kapcsos zárójelek között adhatunk meg, ahol a tulajdonságok neve-érték párokat alkotnak.

```javascript
let szemely = {
    nev: "Kovács Péter", // Tulajdonság név: érték
    kor: 30,
    isTanuló: false
};
```

#### 6. Tömb Literálok (Array Literals)

Tömböket szögletes zárójelek között adhatunk meg, és elemeik vesszővel vannak elválasztva.

```javascript
let szamok = [1, 2, 3, 4, 5]; // Tömb literál
let nevek = ["Anna", "Béla", "Cecil"]; // Tömb literál
```

#### 7. RegEx Literálok (Regular Expression Literals)

Reguláris kifejezéseket perjelek között adhatunk meg, opcionálisan zászlókkal.

```javascript
let regex = /ab+c/i; // Reguláris kifejezés literál, 'i' zászlóval (nem érzékeny kis- és nagybetűkre)
```

### Literálok Használata

A literálokat változókhoz rendelhetjük, függvények argumentumaként használhatjuk, vagy közvetlenül műveletekben is felhasználhatjuk.

```javascript
// Változó deklarálása literálokkal
let szam = 10;
let szoveg = "Ez egy szöveg.";
let igazHamis = true;

// Objektum és tömb létrehozása literálokkal
let autok = [
    { marka: "Toyota", modell: "Corolla" },
    { marka: "Honda", modell: "Civic" }
];

// Függvény hívása literálokkal
console.log("Szám:", szam);
console.log("Szöveg:", szoveg);
console.log("Igaz vagy Hamis:", igazHamis);
```

### Literálok Előnyei

- **Egyszerűség:** Könnyen olvashatók és írhatók közvetlenül a kódban.
- **Tisztaság:** Közvetlen értékeket használnak, ami csökkenti a hibalehetőségeket.
- **Hatékonyság:** Gyorsan feldolgozhatók a JavaScript motorok által.

### Literálok Figyelembe Vétele

- **Immutabilitás:** Bizonyos literálok, mint a számok és sztringek, immutable (változtathatatlan) értékek. Az objektumok és tömbök azonban mutable (változtatható) struktúrák, még akkor is, ha a referenciájuk literálként van megadva.
  
  ```javascript
  let szoveg = "Hello";
  szoveg[0] = "h"; // Nem változtatja meg a sztringet, mert az immutable
  
  let tomb = [1, 2, 3];
  tomb[0] = 99; // A tömb módosul, mert a tömb mutable
  ```

- **Típusok Keveredése:** Egy literál többféle típusú értéket is tartalmazhat, különösen az objektumok és tömbök esetében, amelyek más objektumokat és tömböket is tartalmazhatnak.

  ```javascript
  let vegyes = {
      szam: 42,
      szoveg: "Hello",
      tomb: [true, false, null],
      objektum: { kulcs: "érték" }
  };
  ```

### Összefoglalás

A literálok alapvető elemei a JavaScript nyelvnek, amelyek lehetővé teszik, hogy közvetlenül és egyszerűen megadjuk az adatokat a programban. Megértésük elengedhetetlen a nyelv hatékony használatához és a tiszta, olvasható kód írásához.

## Használati Útmutató

### 1. Dokumentációs Fájl Elhelyezése

Győződj meg róla, hogy a fenti tartalmat egy `.mdx` fájlba mented a `docs` mappán belül. Például, nevezzük el `bevezeto.mdx`-nek.

### 2. Sidebar Konfiguráció

Nyisd meg a `sidebars.js` (vagy `sidebar.js`) fájlt, és győződj meg róla, hogy a `bevezeto` dokumentumot helyesen hivatkozod. Példa:

```javascript
module.exports = {
  docs: [
    {
      type: 'category',
      label: 'Bevezetés',
      items: ['bevezeto'],
    },
    // ... egyéb kategóriák
  ],
};
```

### 3. Projekt Újraépítése

Miután elhelyezted a dokumentációs fájlt és frissítetted a sidebar konfigurációt, építsd újra a Docusaurus projektet:

```bash
npm run build
```

Vagy fejlesztés közben futtasd:

```bash
npm run start
```

### 4. Hibakeresés

Ha bármilyen hibaüzenetet kapsz, győződj meg róla, hogy a Markdown szintaxisa helyes, és nincs benne nem támogatott HTML vagy JSX kód. Használj szintaxis ellenőrzőt vagy lintert a hibák felderítéséhez.
