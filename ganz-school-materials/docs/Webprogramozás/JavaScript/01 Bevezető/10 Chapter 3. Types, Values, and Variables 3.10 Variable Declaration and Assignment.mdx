title: Deklarálás és Értékadás
---

A JavaScript programozási nyelvben a változók használata alapvető fontosságú, mivel ezek tárolják az adatokat, amelyeket a program feldolgoz. A változók deklarálása és értékadása az egyik első lépés, amelyet egy programozó megtanul, mivel ezek nélkül a kód nem lenne képes dinamikusan kezelni és módosítani az adatokat. A változók két fő művelete a *változó deklarálása* és *értékadása*.


## Variable Declaration

- **Variable Declaration (változó deklarálása)**: A változó létrehozása, amely lehetővé teszi, hogy adatokat tároljunk a memóriában. A változó deklarálásához használhatunk kulcsszavakat, mint a `var`, `let`, vagy `const`. 

A **változó deklarálása** azt jelenti, hogy egy új változót hozunk létre, amely egy adott nevet kap. A `let` és `const` a modern JavaScriptben az ajánlott módok, mivel jobban kezelik a blokkszintű hatókört, míg a `var` inkább a régebbi kódokban található meg. A `let` rugalmasabb, mivel újra értéket rendelhetünk hozzá, míg a `const` lehetővé teszi, hogy a változó értéke változhatatlan legyen a későbbiekben.



  Példák:
  ```javascript
  let x;   // A 'x' változó deklarálása, itt még nem adtunk neki értéket.
  ```



## Variable Assignment

- **Variable Assignment (változó értékadása)**: Miután deklaráltuk a változót, értéket rendelhetünk hozzá.

Az **értékadás** azt jelenti, hogy egy konkrét értéket rendelünk egy változóhoz. Az értékadás a `=` operátor segítségével történik. Az értékek lehetnek számok, szövegek, logikai értékek, vagy akár komplex adatstruktúrák is, mint például objektumok vagy tömbök. A változók értéke bármikor módosítható, ha a program logikája ezt megköveteli, kivéve ha `const`-ot használunk, ami biztosítja az érték állandóságát.



  Példa:
  ```javascript
  let x = 5;    // A 'x' változónak értéket adunk: 5
  ```

---

## var

A `var`-t ma már ritkán ajánlják, mivel számos problémát okozhat a globális és függvény szintű hatókör miatt, ami bonyolulttá teszi a kód megértését és hibakeresését. Csak akkor érdemes használni, ha régi kódban dolgozol, vagy ha van valamilyen speciális eset, ahol a `var` viselkedése szükséges.

- **Mikor használjuk?**
  - Ha egy régi JavaScript verzióval vagy régi kódbázissal dolgozol, ami még nem alkalmazza az ES6-ot és újabbakat.

  **Példa**:
  ```javascript
  var globalVar = "Ez egy globális változó";  // A 'var' globális hatókörben működik
  ```
---

## const

A `const` kulcsszó használata ajánlott, ha egy változónak *nem kell módosulnia* a program futása alatt. A `const` garantálja, hogy az értéket egyszer lehet beállítani, és azután nem változtatható meg, így biztonságot nyújt a kódodban, mivel elkerülhetőek a véletlen módosítások.

- **Mikor használjuk?**
  - Ha biztosak vagyunk benne, hogy a változó értéke nem fog változni a program végrehajtása során.
  - Ha objektumokat vagy tömböket használsz, de nem akarod, hogy az objektum vagy tömb maga újra legyen rendelve.
  
  **Példa**:
  ```javascript
  const pi = 3.14;   // Nem változtatható meg
  const person = { name: "Alice", age: 30 };   // Az objektum nem változtatható, de az értékek igen
  ```
---

## let

A `let` kulcsszó akkor használandó, ha a változó értékét a program futása alatt változtatni kell. A `let` lehetővé teszi, hogy a változó értékét többször is módosítsuk, de blokkszintű hatókört biztosít, tehát a változó csak abban a blokkban lesz elérhető, ahol deklarálták.

- **Mikor használjuk?**
  - Ha a változó értékét később módosítani kell.
  - Ha változókra van szükség blokkszintű hatókörrel (pl. egy `if` vagy `for` ciklus belsejében).

  **Példa**:
  ```javascript
  let counter = 0;   // A számláló értéke változhat
  counter++;
  ```

#### Összefoglalva:
- **Használj `const`-ot**, ha biztos vagy benne, hogy a változó értéke nem változik meg.
- **Használj `let`-et**, ha a változó értéke változni fog.
- **Kerüld a `var`-t**, kivéve ha régi kódot kell kezelni, vagy kompatibilitási okok miatt szükséges.

A legjobb gyakorlat az, hogy mindig a **`const`** kulcsszót használd, amikor csak lehetséges, és csak akkor válts **`let`**-re, ha tényleg szükség van a változó módosítására. A `var` helyett a `let` és a `const` biztosítja a jobb kódminőséget és könnyebb hibakeresést.

## Destructuring assignment

A **destructuring assignment** (destrukturálásos értékadás) egy JavaScript-ben elérhető szintaktikai lehetőség, amely lehetővé teszi, hogy egy objektumból vagy tömbből értékeket hozzunk ki, és közvetlenül változókba rendeljük őket. Ez a funkció jelentős mértékben egyszerűsíti a kódot, olvashatóbbá és karbantarthatóbbá teszi, különösen akkor, amikor komplex adatstruktúrákat kell kezelni, például objektumokat vagy tömböket. A destrukturálás segítségével elkerülhetjük a hosszú, többsoros kódot, amelyben egy-egy objektumból vagy tömbből egyes elemeket külön kellene kiválasztanunk.

#### Mi is az a Destructuring Assignment?

A **destructuring assignment** egy olyan mechanizmus, amely lehetővé teszi, hogy egy objektum vagy tömb értékeit közvetlenül változókba rendezzük el, miközben deklaráljuk őket. Ezzel a módszerrel sokkal egyszerűbben érhetünk el adatokat anélkül, hogy külön-külön kellene hivatkozni az egyes elemekre.

A destrukturálás két alapvető formában használható:

1. **Tömbök destrukturálása** – A tömbök elemeit rendelhetjük hozzá változókhoz.
2. **Objektumok destrukturálása** – Az objektumok kulcs-érték párjait rendelhetjük hozzá változókhoz.

## Tömbök destrukturálása

A tömbök destrukturálása esetén a változókat az indexek alapján rendeljük hozzá a tömb elemeihez. Ha például egy tömb első két elemét szeretnénk változókba rendelni, akkor a következő módon használhatjuk a destrukturálást:

**Példa:**
```javascript
let arr = [1, 2, 3];
let [a, b] = arr; // a = 1, b = 2
console.log(a); // 1
console.log(b); // 2
```

Ebben a példában a destrukturálás lehetővé tette, hogy az `arr` tömb első két elemét közvetlenül a `a` és `b` változókba rendeljük.



## Objektumok destrukturálása

Az objektumok destrukturálása hasonló módon működik, de itt a változók a kulcsok alapján kerülnek hozzárendelésre. Az objektum destrukturálása lehetővé teszi, hogy gyorsan hozzáférjünk az objektumban tárolt értékekhez anélkül, hogy hosszú, többsoros kódot írnánk a kulcsok eléréséhez.

**Példa:**
```javascript
let person = { name: "Alice", age: 25 };
let { name, age } = person;
console.log(name); // "Alice"
console.log(age);  // 25
```

Ebben az esetben a `name` és `age` változók közvetlenül a `person` objektum megfelelő kulcsaiból lettek hozzárendelve, anélkül, hogy kézzel kellett volna hozzáférni az objektumhoz.

A destrukturálás során az objektum kulcsnevei meg kell, hogy egyezzenek a változók neveivel, különben hibát kapunk. Azonban lehetőség van arra is, hogy más néven rendelünk hozzá változókat, ha nem akarjuk, hogy a változók nevei pontosan megegyezzenek a kulcsokkal:

**Példa:**
```javascript
let person = { name: "Alice", age: 25 };
let { name: fullName, age: yearsOld } = person;
console.log(fullName); // "Alice"
console.log(yearsOld); // 25
```

A destrukturálás segítségével könnyedén módosíthatjuk, hogy mely kulcsokból származó értékeket rendeljünk hozzá változókhoz, és megadhatjuk az alternatív neveket a változók számára.

## Destructuring Assignment az alapértelmezett értékekkel

A destrukturálásban lehetőség van alapértelmezett értékek megadására is, amelyek akkor kerülnek használatra, ha a desktrukturálás során a kívánt kulcs vagy index nem létezik. Ez különösen hasznos lehet, ha nem vagyunk biztosak abban, hogy a desktrukturált objektumban vagy tömbben mindig szerepelni fog egy adott kulcs vagy érték.

**Példa alapértelmezett értékekkel:**
```javascript
let arr = [1];
let [a, b = 2] = arr; // b alapértelmezett értéke 2, mert arr-ban nincs második elem
console.log(a); // 1
console.log(b); // 2
```

Hasonlóan, objektumoknál is megadhatunk alapértelmezett értéket:

**Példa objektumoknál:**
```javascript
let person = { name: "Alice" };
let { name, age = 30 } = person; // Ha az age nem létezik, alapértelmezett érték 30
console.log(name); // "Alice"
console.log(age);  // 30
```

## Destructuring Assignment alkalmazása függvényeknél

<details className="dropdown-task">

  <summary>
  <strong>destrukturálás függvényeknél - Kitekintés</strong>
  
  A destrukturálás különösen hasznos lehet, amikor egy függvény több paramétert vár, és a bemeneti adatokat objektumok vagy tömbök formájában kapja meg. Így a destrukturálás segíthet egyszerűsíteni a paraméterek kezelését és a kód olvashatóságát.
  </summary>


**Példa függvényeknél:**
```javascript
function greet({ name, age }) {
    console.log(`Hello, my name is ${name} and I am ${age} years old.`);
}

let person = { name: "Alice", age: 25 };
greet(person);  // "Hello, my name is Alice and I am 25 years old."
```

Itt a `greet` függvény egy objektumot vár, és közvetlenül a destrukturálás segítségével hozzáfér a `name` és `age` tulajdonságokhoz.

</details>


A **destructuring assignment** egy rendkívül hasznos eszköz a JavaScriptben, amely lehetővé teszi, hogy egyszerű, tiszta és olvasható módon dolgozzunk komplex adatstruktúrákkal, például tömbökkel és objektumokkal. A destrukturálás különösen akkor hasznos, amikor nagyobb adatszerkezeteket kell manipulálni, például API válaszok feldolgozásakor vagy funkcióparaméterek kezelésénél.

        const [karakter1, karakter2, karakter3, karakter4] = karakterek;

            valasztottKarakter = karakter4;
