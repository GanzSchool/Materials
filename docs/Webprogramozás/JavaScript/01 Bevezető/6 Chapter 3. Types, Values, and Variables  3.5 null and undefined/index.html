A null és undefined dilemmája

Mi az a null és undefined?​

A null és undefined két alapvető érték a JavaScript-ben, amelyeket gyakran használunk az értékek hiányának vagy nem létezésének kifejezésére. Bár mindkét érték a hiányzó vagy nem elérhető értékek jelzésére szolgál, jelentésük és használatuk különbözik.

A null egy speciális érték a JavaScript-ben, amely azt jelzi, hogy egy változó szándékosan nem rendelkezik értékkel. A null érték kifejezi a hiányzó objektumot vagy nem létező értéket. A programozó akkor használja, amikor egy változót egyértelműen üresnek vagy "nincs értéke"-nek kíván beállítani, például ha egy változó vagy objektum értékét később szeretné inicializálni vagy meghatározni. Fontos, hogy a null különbözik az undefined-tól, mivel az előbbi kifejezetten szándékos értékhiányt jelent, míg az utóbbi azt jelzi, hogy a változó nincs inicializálva vagy nincs hozzárendelve érték.

Az undefined egy speciális érték a JavaScript-ben, amely azt jelzi, hogy egy változónak nincs hozzárendelve érték. Ez akkor történik meg, ha egy változó deklarálásra kerül, de még nincs inicializálva (nem kapott értéket), vagy ha például egy függvény nem ad vissza semmilyen értéket. Az undefined az alapértelmezett érték, amelyet a JavaScript automatikusan rendel a nem inicializált változókhoz, illetve ha egy objektum nem tartalmaz egy adott tulajdonságot, akkor annak az értéke undefined.

Különbség a null és az undefined között:​

null:

  • A null kifejezett érték, amelyet a programozó ad a változónak. Az null azt jelenti, hogy a változónak nincs értéke, és ez egy szándékos döntés a programozó részéről.
  • Az értéket explicit módon kell beállítani.

undefined:

  • Az undefined az alapértelmezett érték, amelyet a JavaScript rendel egy változóhoz, ha az nincs inicializálva, vagy egy függvény nem ad vissza semmilyen értéket.
  • Az értéket automatikusan adja a JavaScript.

Hasonlóságok:​

  • Mind a null, mind az undefined az értékek hiányát jelzik.
  • Falsy értékek: Mindkét érték falsy érték, ami azt jelenti, hogy ha a logikai kifejezésekben (pl. if feltételekben) használjuk őket, akkor mindkettő hamisnak számít. Például:
if (null) {
    console.log("Ez nem fog lefutni, mert null egy falsy érték.");
}

if (undefined) {
    console.log("Ez sem fog lefutni, mert undefined is falsy érték.");
}
  • == (egyenlőség operátor): Az == operátor összehasonlítja a null és undefined értékeket, és igaznak tekinti őket, tehát:
console.log(null == undefined);  // true
  • === (szigorú egyenlőség operátor): Az === operátor viszont különbözőnek tekinti őket, mert null és undefined nem ugyanaz a típus.
console.log(null === undefined);  // false

Null​

A null egy kifejezetten üres vagy nem létező értéket jelöl. Ezt az értéket általában akkor használjuk, amikor valamilyen változónak nincs értéke, és egyértelműen jelölni szeretnénk, hogy nem egy objektum vagy nem egy érvényes adat szerepel ott. A null tehát egy explicit üres érték.

  • Típusa: object (ez egy ismert JavaScript hiba, amely nem változott, de fontos tudni)
  • Jelentése: "Semmi" vagy "nem létező érték"

Példa:

let valami = null;
console.log(valami); // null

Használat:

  • Ha egy változó nem rendelkezik értékkel, és azt explicit módon szeretnénk jelezni, hogy az érték hiányzik, használhatjuk a null-t.

Példa:

let felhasznalo = null; // A felhasználó nincs bejelentkezve

Undefined​

Az undefined azt jelenti, hogy egy változó még nem kapott értéket, vagy nem lett definiálva. Az undefined automatikusan keletkezik, ha egy változó deklarálva lett, de nem lett neki érték hozzárendelve. Az undefined ugyanúgy használható akkor is, amikor egy objektumban nem található meg a kívánt kulcs.

  • Típusa: undefined
  • Jelentése: "Nem definiált" vagy "nem értékelt változó"

Példa:

let valami;
console.log(valami); // undefined

Használat:

  • Ha egy változó deklarálva van, de még nincs értéke, automatikusan undefined-ra van állítva.
  • Ha egy függvény nem ad vissza értéket, akkor az is undefined-ot ad vissza.

Példa:

let felhasznalo;
console.log(felhasznalo); // undefined

Ha például egy függvény nem tartalmaz return utasítást, akkor a visszatérési értéke undefined lesz:

function nincsVisszatérésiÉrték() {}
console.log(nincsVisszatérésiÉrték()); // undefined

Különbségek​

  • null egy érték, amit az ember kifejezetten rendel egy változóhoz, hogy jelezze, hogy az változó üres vagy nincs értéke.
  • undefined akkor jelenik meg, amikor egy változó nincs inicializálva vagy nem lett definiálva egy értékkel.

Összefoglaló táblázat​

Összegzés​

  • Használjuk a null-t, amikor egy változó szándékosan nem rendelkezik értékkel.
  • Az undefined akkor jelenik meg, ha a változó nem lett inicializálva, vagy ha egy függvény nem ad vissza értéket.

Gyakorlati példák​

Felhasználó bejelentkezésének állapota

Példa:​

let felhasznalo = null; // A felhasználó nincs bejelentkezve

// Később a bejelentkezés után változik az értéke
felhasznalo = {
    nev: "Kovács Péter",
    email: "peter.kovacs@example.com"
};

if (felhasznalo === null) {
    console.log("A felhasználó nincs bejelentkezve.");
} else {
    console.log("Üdvözöljük, " + felhasznalo.nev);
}

A felhasznalo változó kezdetben null értéket kap, jelezve, hogy a felhasználó nincs bejelentkezve. Később, ha a felhasználó bejelentkezik, a változó egy objektumot kap, amely tartalmazza a felhasználó nevét és email címét.

Változók deklarálása érték nélkül

Példa:​

let felhasznalo;
console.log(felhasznalo);  // undefined

// Később értéket rendelünk hozzá
felhasznalo = "Kovács Péter";
console.log(felhasznalo);  // Kovács Péter

A felhasznalo változó deklarálva van, de nincs hozzárendelt értéke, ezért automatikusan undefined értéket kap.

Objektum attribútum hiánya
let szemely = {
    nev: "Kovács Péter",
    kor: 30
};

console.log(szemely.cim); // undefined, mert nincs "cim" attribútum

Ha egy objektumban olyan kulcsot keresünk, amely nem létezik, akkor a válasz undefined lesz. Ebben az esetben a cim kulcs nem létezik a szemely objektumban.

Gyakorlati feladatok​

Adatok bekérése - prompt() gyorstalpaló

Mi a prompt()?​

A prompt() egy beépített JavaScript függvény, amely egy felugró ablakot hoz létre a böngészőben, és lehetővé teszi a felhasználó számára, hogy adatot adjon meg. A felhasználó beírt szövege aztán visszaadásra kerül a program számára, hogy további műveleteket végezhessünk vele.

Alap szintaxis:​

let valtozo = prompt("Kérlek, írd be a kérdésre a választ:");
  • A prompt() egy kérdést vagy szöveget jelenít meg, amely megjelenik a felhasználónak a felugró ablakban.
  • Az a szöveg, amit a felhasználó beír, visszaadásra kerül és eltárolódik a valtozo változóban.

Példák​

Egyszerű adatbekérés:​

let nev = prompt("Mi a neved?");
console.log("Szia, " + nev + "!");

🔹 Mi történik?

  • A prompt() felugró ablakban megjelenik a kérdés: "Mi a neved?"
  • A felhasználó válaszát eltároljuk a nev változóban, majd a válasz szerint köszönünk a konzolon.

Szám adat bekérése:​

A prompt() mindig szöveges adatot ad vissza, függetlenül attól, hogy a felhasználó számot ír. Ha számot szeretnél kezelni, szám típusra kell konvertálnod.

let eletkor = prompt("Hány éves vagy?");
eletkor = Number(eletkor); // Átalakítjuk számra

if (eletkor >= 18) {
    console.log("Nagykorú vagy.");
} else {
    console.log("Még kiskorú vagy.");
}

🔹 Mi történik?

  • A prompt() kérdése után a felhasználó beírja az életkorát, ami szöveges formában érkezik.
  • A Number() függvénnyel számra konvertáljuk.
  • Az életkort kiértékeljük, és kiírjuk, hogy nagykorú-e a felhasználó.

A prompt() visszatérési értéke:​

  • Ha a felhasználó megerősíti az adatot a "OK" gombbal, akkor a beírt szöveget kapod vissza.
  • Ha a felhasználó nem ad meg semmit (csak OK-t nyom), az érték üres sztring ("").
  • Ha a felhasználó Mégse gombot nyom, az érték null lesz.

Üres adat kezelése:​

let varos = prompt("Melyik városban élsz?");
if (varos === null || varos.trim() === "") {
    console.log("Nem adtál meg várost.");
} else {
    console.log("Te " + varos + " városban élsz.");
}

🔹 Mi történik?

  • A varos változóba eltároljuk a felhasználó válaszát. Ha nem ír be semmit, vagy rákattint a Mégse gombra (ami null), akkor kiírjuk, hogy nem adott meg várost.
  • Ha van válasz, akkor a város nevét kiírjuk.

Hasznos dolgok, amiket tudnod érdemes:​

  • Mindig szöveg: A prompt() mindig szöveget ad vissza. Ha számot szeretnél kezelni, akkor használd a Number() függvényt a konvertáláshoz.

  • Üres string: Ha a felhasználó nem ad meg semmit és rákattint az OK gombra, az érték üres string lesz (""), amit kezelni kell, ha szükséges.

  • null: Ha a felhasználó a Mégse gombra kattint, akkor az érték null lesz.

Összegzés:​

A prompt() segít egyszerű adatbekérésben a felhasználótól. Az adat mindig szöveges típusú, és az értéket könnyedén eltárolhatod változókban, hogy a továbbiakban felhasználhasd őket.

null​

Felhasználó életkora – Hiányzó adat kezelése
  • Hozz létre egy age nevű változót, amely a felhasználó életkorát tárolja.
  • Ha a felhasználó nem adott meg életkort, akkor az értéke legyen null.
  • Ha az életkor null, írj ki egy üzenetet: "Az életkor nincs megadva."
  • Ha az életkor nem null, akkor írd ki: "A felhasználó életkora: X", ahol X az életkor értéke
let age = 56;

if (age === null) {
    console.log("Az életkor nincs megadva.");
} else {
    console.log(`A felhasználó életkora: ${age}`);
}
Bejelentkezés kezelése
  • Hozz létre egy loggedInUser nevű változót.
  • Ha a felhasználó nincs bejelentkezve, az értéke legyen null.
  • Ha nincs bejelentkezve, írj ki egy üzenetet: "Nincs bejelentkezett felhasználó."
  • Ha bejelentkezik (pl. "Anna" néven), frissítsd a változót és írd ki: "Üdv újra, Anna!"
let loggedInUser = null;

if (loggedInUser === null) {
    console.log("Nincs bejelentkezett felhasználó.");
} else {
    console.log("Üdv, " + loggedInUser + "!");
}

// Bejelentkezés után
loggedInUser = "Anna";

if (loggedInUser !== null) {
    console.log("Üdv újra, " + loggedInUser + "!");
}

undefined​

Termék ára – Hiányzó érték figyelése
  • Hozz létre egy productPrice nevű változót, amely egy termék árát tárolja.
  • Ha nincs beállítva érték (azaz undefined), írj ki egy üzenetet: "A termék ára nincs megadva."
  • Ha az ár meg van adva, írd ki: "A termék ára: X Ft", ahol X az ár értéke.
let productPrice;

if (productPrice === undefined) {
    console.log("A termék ára nincs megadva.");
} else {
    console.log("A termék ára: " + productPrice + " Ft");
}
Profil beállításai – Hiányzó tulajdonságok kezelése
  • Hozz létre egy userProfile nevű objektumot, amely egy name tulajdonságot tartalmaz.
  • Ellenőrizd, hogy van-e email tulajdonsága az objektumnak.
  • Ha nincs email tulajdonság (azaz az undefined), írj ki egy üzenetet: "Az e-mail cím nincs beállítva."
  • Ha van email, írd ki: "Felhasználó e-mail címe: X" (ahol X az e-mail címe).
let userProfile = {
    name: "Zoltán"
};

if (userProfile.email === undefined) {
    console.log("Az e-mail cím nincs beállítva.");
} else {
    console.log("Felhasználó e-mail címe:", userProfile.email);
}
